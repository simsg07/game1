Pygame 실습 기록: 2인용 성장 대전 게임 구현
    1. 주요 질의응답 및 실험 (Q&A)

Q1: screen.fill(WHITE) 줄을 삭제하면 어떻게 되는가?

    AI 답변: fill()은 매 프레임마다 화면을 깨끗이 지우는 '지우개' 역할이다. 삭제하면 이전 프레임의 그림이 지워지지 않고 계속 덧칠되어 잔상이 남게 된다.

    적용 결과: 원이 움직일 때 길게 늘어지는 선처럼 잔상이 남는 것을 확인. 애니메이션에서 '화면 갱신'의 중요성을 이해함.

    Q2: clock.tick(60)을 5로 낮추면 어떻게 되는가?

    AI 답변: tick() 안의 숫자는 초당 프레임 수(FPS)를 뜻한다. 숫자가 낮아지면 1초에 화면을 그리는 횟수가 줄어들어 화면이 뚝뚝 끊겨 보이게 된다.

    적용 결과: 화면이 매우 버벅거리고 조작 반응이 느려지는 '렉(Lag)' 현상을 체감함.

Q3: 원이 화면 밖으로 나갔을 때 반대편에서 나타나게 하려면?

    AI 답변: 조건문(if)을 사용하여 좌표가 화면 경계(800, 600)를 넘었는지 체크한 뒤, 좌표를 반대편 끝으로 강제 재설정(Teleport)하는 '화면 루핑' 로직을 적용한다.

    적용 결과: 원이 화면 끝으로 사라지면 반대편에서 자연스럽게 튀어나오는 무한 맵 로직을 성공적으로 구현함.

Q4: 두 플레이어를 각각 조작하고 아이템 점수 시스템을 만들려면?

    AI 답변: 각각의 좌표 변수(x1, y1 / x2, y2)를 만들고 화살표 키와 WASD 키를 나누어 매핑한다. 아이템과의 거리를 계산해 충돌 시 점수를 올리고 위치를 랜덤하게 바꾼다.

    적용 결과: 5점 선취 시 승리하는 2인용 대결 게임 완성. 아이템 획득 시 플레이어의 크기가 커지는 시각적 효과 추가.

    2. 어려웠던 점 및 해결 (Reflection)

    좌표계의 이해: 수학과 달리 Y축이 아래로 갈수록 커지는 컴퓨터 그래픽 좌표계가 생소했으나, 직접 코드를 수정하며 방향 감각을 익힘.

    초기 화면 버그: 실행 시 창이 잠시 반투명하게 뜨는 현상을 set_mode 직후 fill과 flip을 실행해 주는 초기화 순서 조정으로 해결함.

3. 배운 점 (Key Takeaways)
화면 루핑(Looping): 좌표 경계값을 체크하여 객체를 반대편으로 이동시키는 게임적 허용 기법을 익힘.

상태 제어: game_over 변수를 활용해 게임의 진행과 멈춤을 제어하고, 'R' 키를 활용한 다시 시작(Restart) 기능을 통해 게임 루프의 구조를 깊이 이해함.
