4주차: Pygame 충돌 판정 알고리즘 실습

1. 실습 내용 및 관찰

    시각화: Circle, AABB, OBB의 충돌 범위를 화면에 
    직접 그려 정밀도 차이 분석

    원형 vs AABB: 원형은 회전에 강하지만 긴 물체에 
    부적합하고, AABB는 계산이 빠르나 회전 시 판정 
    박스가 비대해지는 오차 확인

    AABB vs OBB: OBB(SAT)는 계산량이 많지만, 
    회전하는 물체의 테두리에 완벽히 밀착되어 
    시각적으로 가장 정확한 충돌 감지 가능

2. 주요 Q&A 요약

    성능 최적화: 거리 계산 시 math.sqrt 대신 
    제곱 
    비교법($d^2 < (r1+r2)^2$)을 사용하여 
    불필요한 
    연산 부하를 줄이는 원리 파악

    회전 대응: Pygame의 Rect는 축 고정 방식이라 
    회전 
    시 오차가 큼. 이를 해결하기 위해 **SAT
    (분리축 정리)**를 도입, 물체 사이에 '틈'이 있는지 
    판단하는 논리 구현

    시각적 피드백: 그리드 배경과 충돌 시 색상 
    변화를 
    적용해 로직의 정상 작동 여부를 직관적으로 
    확인하는 디버깅 환경 구축

    3. 시행착오 및 해결
    벡터 내적 활용: 수학으로만 접한 내적이 
    '그림자 투영'을 통한 충돌 판정에 쓰이는 
    실용적 사례를 경험함

    좌표 추적: 회전하는 객체의 4개 꼭짓점을 
    구하기 위해 pygame.Vector2와 삼각함수를 
    연동하여 정밀한 좌표 갱신 성공

4. 결론 및 배운 점

    가벼운 Circle/AABB로 먼저 거르고 필요한 경우에만 
    무거운 OBB를 호출하는 단계별 판정 최적화의 
    중요성을 체감했습니다. 추상적인 기하학 공식을 
    실제 코드로 구현하며 게임 수학의 실용성을 
    깊이 이해했고 sprites.py 모듈화를 통해 자산 관리와 
    로직을 분리하는 설계 감각을 익혔습니다
    향후 프로젝트에서도 성능과 정밀도의 밸런스를 고려한 
    충돌 시스템을 구축할 계획입니다
